# When to create a skill

Create a skill when a repeated class of work needs a concise, reusable instruction set that a new agent cannot safely infer from the repository alone. The skill should define durable conventions, boundaries, and references that make the work more consistent across human-started sessions.

Do not create a skill for a one-off task, a temporary handoff, or a need to automate another host. Prefer ordinary repository documentation when the guidance is specific to one project or can be understood directly from the task artifact. A candidate skill belongs in this repository only when it remains static, host-neutral, human-mediated, and passes every scope-admission test in `agent-skill/references/protocol.md`.
